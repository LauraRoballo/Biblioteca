# Requerimientos no funcionales 🔧

## RN-001: Rendimiento del sistema
El sistema debe garantizar una respuesta rápida ante las solicitudes de los usuarios, mostrando resultados de búsqueda, inicio de sesión, o consultas en menos de 3 segundos

## RN-002: Seguridad y autenticación 
El sistema debe contar con un proceso de autenticación seguro mediante usuario y contraseña institucional, con niveles de acceso definidos según el rol (administrador, bibliotecario, docente o alumno).

## RN-003: Usabilidad e interfaz 
La interfaz del sistema debe ser intuitiva, clara y fácil de usar, permitiendo que cualquier usuario del colegio pueda navegar sin requerir capacitación técnica avanzada.

## RN-004: Disponibilidad del sistema 
El sistema debe estar disponible durante el horario de funcionamiento de la institución educativa (de 6:00 a.m. a 6:00 p.m.) sin interrupciones prolongadas.

## RN-005: Mantenimiento y escalabilidad 
El sistema debe estar diseñado para permitir futuras actualizaciones, mejoras o ampliaciones de módulos sin afectar la información almacenada.

## RN-006: Copias de seguridad 
El sistema debe generar copias de seguridad automáticas de la base de datos de forma semanal para evitar la pérdida de información en caso de fallos o errores.

## RN-007: Compatibilidad y portabilidad 
El sistema debe ser compatible con los principales navegadores web (Chrome, Edge, Firefox) y adaptable a diferentes dispositivos (computadores y tabletas).
